A quick scan of the dataset –

Questions to ask your data
Are there missing or truncated values?
Are there out of range values?
Look at all the values that appear in a column/row – do they make sense?
Are there similar repeated values?
Is the data format what you would expect for the data type?
How was the data collected? Is it from a survey (survey – online survey vs. phone calls)?
How was the data verified? If data is entered by hand by a human person, how was it verified?
When was the data collected?
What is the gap between being collected and published? ie., there will always be a lag between when census data is collected from Canadians and when it is made available (to download/access)

Check for skewed data

  • Inflation
  • Currency inflation means that over time money changes in value
  • Check with the source of the data to see if the numbers are adjusted
  • Adjust for inflation — the Bank of Canada’s inflation calculator)

    • check if the data has been “seasonally adjusted”
  • Timeframe or frame of reference has been manipulated

Check for missing values or zeros/arbitrary values replace missing values

  • Blank / “null” values
    • Check for blank or “null” values
    • Confirm what blank or “null” values mean (ie., is the value unavailable, missing, etc.)
    • Data is missing
      • ie., if a dataset is supposed to be about “Canada,” check that all 10 provinces and 3 territories are reflected in the data
      • do a first pass on a dataset to make sure the data provided matches what you expected
  • When in doubt: Go back to the source of the data to confirm why a value is missing, zero (0), or null

If a data set is to represent a full year, check that all months are recorded

Check for units, categories, and labels

  • Currency
    • Check if foreign prices may be specified in their local currency
      • ie., is you data in USD or CAD? 🔗 RBC calculator
    • What year is the dollar value from
      • ie., a $100 in 2010 is not $100 in 2019
      • Check and adjust for inflation 🔗 Bank of Canada calculator
  • Categories
    • Bad categories can artificially exclude data.
    • For example: crime statistics. A bad or changing definition might mean a crime is counted in a different category than you expect or that it wasn’t counted at all.
  • Labels
    • Ambiguity = a likely error
    • Confirm with the source of the dataset your understanding matches their definitions

Check for inconsistencies

  • Spelling is inconsistent
    • Check that location/place names are consistent
      • ie., Los Angelos instead of Los Angeles
    • Check locations for inconsistent spelling, or inconsistent short forms/abbreviations
      • ie., both Sault Sainte. Marie and Sault Ste. Marie
    • Often indicates that the data was compiled by a human person, reason to be skeptical of the data
    • You will need to manually correct mistakes or otherwise account for them in your analysis and reporting
    • Be sure to document your changes

Check for numbers as text and text as numbers

  • Not all numerals are numbers
  • Spreadsheets programs often make the mistake of assuming numerals are numbers and stripping the leading zeros.
    • ie., New England zip codes almost always start with zero, i.e. 02901
    • Excel assumes it’s numeric and “helpfully” drops that leading zero, making the value inaccurate
  • Good practice is to store numbers without formatting and to include supporting information in column names or metadata.

Zeros replace missing values / data is missing

  • If you see zeros in a series of numbers you should ask yourself if those values are really the number 0 or if they instead mean "nothing". (-1 is also sometimes used this way.)

Check number of columns and rows

  • Excel has (had) a maximum of 65,536 rows. If your dataset has 65,536 rows, you likely have a portion of the data. Go back to the source to confirm or request the full dataset.

  • Apple’s Numbers can only handle 255 columns. If 255 rows, you might only have a portion of the data. Go back to the source to confirm or request the full dataset.

  • 💻To fix: Confirm by opening the CSV in Sublime Text or Atom that there are more than the maximum number of values.

Check for these suspicious values

If you see any of these values in your data, treat them with an abundance of caution:

Numbers:

  • 65,535
  • 255
  • 2,147,483,647
  • 4,294,967,295
  • 555-3485
  • 99999 (or any other long sequence of 9’s)
  • 00000 (or any other sequence of 0’s)

Dates:

  • 1970-01-01T00:00:00Z
  • 1969-12-31T23:59:59Z
  • January 1st, 1900
  • January 1st, 1904

Spreadsheet has dates in 1900, 1904, 1969, or 1970

Excel’s default date from which it counts all other dates is January 1st, 1900, unless you’re using Excel on a Mac, in which case it’s January 1st, 1904. There are a variety of ways in which data in Excel can be entered or calculated incorrectly and end up as one of these two dates. If you spot them in your data, it’s probably an issue.

Many databases and applications will often generate a date of 1970-01-01T00:00:00Z or 1969-12-31T23:59:59Z which is the Unix epoch for timestamps. In other words this is what happens when a system tries to display an empty value or a 0 value as a date.

Locations:

  • 0°00'00.0"N+0°00'00.0"E or simply 0°N 0°E
  • US zip code 12345 (Schenectady, New York)
  • US zip code 90210 (Beverly Hills, CA)

Each of these numbers has an indication of a particular error made by either a human or a computer. If you see them, ensure they actually mean what you think they mean!
